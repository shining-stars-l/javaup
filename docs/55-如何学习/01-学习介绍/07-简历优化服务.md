# 简历优化服务

## 简历的重要性

简历是求职的第一张名片，也是获得面试机会的敲门砖。HR每天要筛选大量简历，平均看一份简历的时间不超过30秒。在这短短的时间里，一份结构清晰、亮点突出的简历才能脱颖而出。

但是，找我咨询的很多小伙伴在写简历时，或多或少都会存在以下这些问题：

:::warning 常见简历问题
- **专业技能太简单**：只写"熟悉Java"、"了解Redis"，没有具体技术点支撑
- **项目经历像流水账**：全是业务描述，缺乏技术栈应用和量化数据
- **排版混乱**：中英文不规范、大小写不一致、格式杂乱
- **技术栈过时**：还在写JSP、Struts等已经淘汰的技术
- **重点内容不突出**：把Idea工具、Git的使用放在最前面
:::

## 如何提交简历修改？

### 第一步：准备材料

除了简历本身，还需要提供一段个人介绍，帮助更快了解你的情况。建议包含以下信息：

- 学历背景（本科/硕士/专科，学校层次）
- 目前状态（在校/实习/工作）
- 求职目标（校招/社招，期望公司类型）
- 技术栈方向（后端/前端/全栈）
- 存在的困惑（简历没有亮点、不知道怎么写项目等）

### 第二步：发送邮件

将简历作为附件，个人介绍作为正文内容，发送到指定邮箱。

详细说明请参考：[简历修改说明](https://articles.zsxq.com/id_8akdj30btvdz.html)

### 第三步：等待反馈

通常24小时内会收到修改建议，修改后可以继续提交，直到满意为止。

## 专业技能怎么写？

很多人的专业技能写得过于笼统，缺乏说服力。正确的写法应该是：**无序列表 + 主题 + 掌握程度 + 具体知识点**

### 反面示例

> 熟悉Java基础，了解常用集合，熟悉多线程

这种写法过于空洞，看不出你到底掌握了什么。

### 正确写法

> - **Java基础**：掌握 Java 基础知识，如集合框架、I/O流、Stream 流、反射机制等，阅读过 HashMap 源码，了解其扩容机制；熟悉 Java 开发中常见的设计模式，如工厂模式、代理模式、单例模式等。
> 
> - **并发编程**：熟练使用 Java 中的并发容器，如 ConcurrentHashMap、CopyOnWriteArrayList 等，对线程池、synchronized 关键字、volatile 关键字、AQS、ThreadLocal 等有深入的理解。
> 
> - **JVM**：熟悉 Java 虚拟机，对 JMM、GC 底层算法、常见垃圾回收器较为熟悉，熟练掌握 JVM 调优并能在工作中加以运用，对 Class 文件、类加载机制作过深入研究。
> 
> - **Redis**：熟悉 Redis 的常见数据类型和应用场景，以及持久化、过期淘汰策略、分布式锁等特性，能够解决高并发下的缓存穿透、雪崩、击穿问题。

## 项目经历怎么写？

项目经历是面试官最关注的部分，很多面试都是围绕项目展开。

### 反面示例

只写业务描述，没有技术栈应用：

> 负责用户管理模块的开发，实现了用户注册、登录、信息修改等功能。

### 正确写法

技术栈 + 业务 + 量化数据：

> - 使用 FastExcel 实现 PUPV 数据的批量导出功能，500 万条数据导出仅需 1 秒，并结合自定义线程池 + CountDownLatch 进行并发处理，导出性能提升近 60 倍。
> 
> - 集成本地缓存 Guava 和 Caffeine，有效提高服务的吞吐率，经实测在 2核4G 的情况下能达到 1200 QPS。
> 
> - 利用 RestHighLevelClient 接入 ElasticSearch 的全文搜索能力，支持关键词高亮、模糊查询及搜索自动补全关键字，检索效率提升了 60%。

### 项目经历模板

一个完整的项目经历应该包含：

1. **项目概述**：用一两句话说明项目是做什么的
2. **技术栈**：列出项目使用的主要技术
3. **个人职责**：负责哪些模块，是主要负责人还是协作
4. **技术亮点**：解决了什么问题，使用了什么技术方案
5. **成果量化**：性能提升了多少，解决了什么问题
