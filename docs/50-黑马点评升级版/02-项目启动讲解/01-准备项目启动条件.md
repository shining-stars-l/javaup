# 准备项目启动条件

:::info plus 版本专属
此章节是黑马点评 Plus 版本中专有的内容，而在整套文档中将普通版本和 Plus 版本都融合在了一起，让大家更方便的学习。
:::

## 项目下载
### 项目地址：
[https://gitee.com/java-up-up/hmdp-plus](https://gitee.com/java-up-up/hmdp-plus)

### 项目HTTPS克隆地址：
```plain
git clone https://gitee.com/java-up-up/hmdp-plus.git
```

### 项目SSH克隆地址：
```plain
git clone git@gitee.com:java-up-up/hmdp-plus.git
```

### 克隆项目注意：
要使用git clone使用上述的HTTPS或者SSH地址克隆下载项目，不要下载ZIP，否则不会通过git关联到项目的远程仓库，不能够git pull更新项目

## IntelliJ IDEA导入项目
知道了项目地址后，使用IntelliJ IDEA将项目导入

<img src="/img/hmdp-plus/启动/WPS拼图0.png" alt="表关系" width="70%" />

通过 IntelliJ IDEA将项目导入，URL为项目的https或者ssh地址，Directory为项目的目录，当项目克隆下来后，等待IntelliJ IDEA通过maven将项目构建完毕

### 项目目录注意：
项目所在的目录中**不要有中文！** 必须是全英文的，否则在服务启动时会出现class找不到的问题！

为了加快maven的构建速度，小伙伴可在自己电脑的maven中配置阿里云的镜像仓库

#### 文件位置
```plain
apache-maven-3.6.3\conf\settings.xml
```

#### 阿里云仓库配置
```xml
<mirrors>
    <mirror>
      <id>aliyunmaven</id>
      <mirrorOf>*</mirrorOf>
      <name>阿里云公共仓库</name>
      <url>https://maven.aliyun.com/repository/public</url>
    </mirror>
</mirrors>
```

当项目构建完毕后，可在项目根目录下执行

```shell
mvn clean compile
```

编译一下项目是否有错误

## 导入数据库表和数据
sql文件分为**创建数据库、创建表和数据**

### sql文件位置
<img src="/img/hmdp-plus/启动/sql文件位置.png" alt="表关系" width="35%" />

### 如何执行sql文件
1. 先执行 **1_create_database.sql** 创建好数据库
2. 再执行 **hmdp_0.sql** 和 **hmdp_1.sql** 创建好表和数据

### 注意：
如果直接执行sql文件报错的话，可以把sql文件中的语句粘贴到图形化工具的控制台中执行
