# 组 维度的服务过滤功能

## 组的作用
- 使用不同组：来实现对服务进行逻辑上的分组，主要作用在同一个区域下的不同组的服务之间实现路由隔离。

- 不支持降级组的策略。
## 组的设置
在配置中指定开启或关闭（默认关闭状态）
```yaml
spring:
  application:
    link-flow:
      work:
        # 组规则隔离的开关
        group:
          activate: true
```
具体服务自己的区域值通过在项目的配置文件中指定
```yaml
spring:
  cloud:
    link-flow:
      metadata:
        group: test-group
```
在项目启动后，就会自动将组放置在服务中的元数据中
